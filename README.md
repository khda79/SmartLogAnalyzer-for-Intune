# SmartLogAnalyzer for Intune

SmartLogAnalyzer for Intune is a Windows desktop tool for reading Microsoft Intune
Device Diagnostics ZIP files and local device diagnostic captures.

It helps Intune and endpoint administrators turn raw diagnostic exports into a
clear view of device join state, MDM enrollment, IME errors, Windows Update
signals, hardware/security posture, health findings, and a self-contained HTML
report.

Built by [WorkplaceCloudHub](https://workplacecloudhub.com).

## Highlights

- Analyze Intune Device Diagnostics ZIP files downloaded from the Intune portal.
- Run a local device scan when the ZIP is not available.
- Parse DSRegCmd status, AAD/Entra join, PRT, WAM, tenant and TPM details.
- Read MDM enrollment registry exports under `HKLM\Software\Microsoft\Enrollments`.
- Parse MDM Diagnostics CAB/HTML reports generated by `MDMDiagnosticsTool.exe`.
- Detect IME/SCCM log errors by log theme.
- Parse Windows Update registry, policy, ReportingEvents and ETL-derived signals.
- Parse Windows Event Logs from EVTX files.
- Report firewall, battery, certificates, installed apps, drivers, WiFi and Autopatch data.
- Run local health checks for BitLocker, Defender, storage, performance and Entra join.
- Generate a local "Top 5 actions" view, health score, root cause hints and unified timeline.
- Search across local insights, root causes, WUfB settings and timeline events.
- Export an anonymized copy of a diagnostic ZIP for support sharing.
- Export a standalone HTML report with no external dependencies.
- Optionally use Claude, OpenAI or local Ollama for AI-assisted diagnostics.

## Requirements

### Run the compiled executable

- Windows 10 or Windows 11.
- No Python installation required.
- Administrator rights are recommended for local device collection.

Download `SmartLogAnalyzer.exe` from the [GitHub Releases](../../releases) page.

### Run from source

- Windows 10 or Windows 11.
- Python 3.10 or later.
- Tkinter, included with standard Python on Windows.
- Dependencies from `requirements.txt`.

```bash
pip install -r requirements.txt
python SmartLogAnalyzer.py
```

## Usage

### Analyze an Intune diagnostics ZIP

1. In the Intune admin center, open the device.
2. Go to `Devices > Device diagnostics > Download`.
3. Open SmartLogAnalyzer.
4. Select `Analyze Intune DiagLogs ZIP`.
5. Browse the analysis tabs.
6. Use `Export HTML Report` when you need a portable report.
7. Use `Export anonymized ZIP` when you need to share a redacted diagnostic archive.

### Analyze the local device

1. Run SmartLogAnalyzer as administrator.
2. Select `Analyze local device`.
3. Choose where to save the generated diagnostic ZIP.
4. The app collects local logs, registry exports, event logs and health data.
5. The generated ZIP is analyzed automatically.

## Privacy And AI

The core analysis runs locally.

When using the AI tab, parsed diagnostic text is sent to the selected provider:

- Claude via Anthropic API.
- OpenAI via OpenAI API.
- Ollama locally via the configured Ollama URL.

API keys are entered in the app UI. By default, API keys are not stored. If
`Remember API key on this device` is checked, the key is saved locally in:

```text
%USERPROFILE%\.smartloganalyzer_ai.json
```

Leave that option unchecked on shared, admin or jump-box workstations.

## Release And Signing

GitHub Actions builds the executable on every push and pull request.

Signing and GitHub Release publication run only when a version tag matching
`v*.*.*` is pushed:

```bash
git tag v2.1.1
git push origin v2.1.1
```

Tagged builds:

1. Build `SmartLogAnalyzer.exe` on `windows-latest`.
2. Upload the unsigned executable as a workflow artifact.
3. Submit the artifact to SignPath.
4. Poll SignPath until signing completes.
5. Download the signed executable.
6. Publish the signed executable to GitHub Releases.

Required GitHub configuration:

- Repository or organization variable: `SIGNPATH_ORG_ID`.
- Repository or organization secret: `SIGNPATH_API_TOKEN`.
- SignPath project slug: `smartloganalyzer-intune`.
- SignPath signing policy slug: `release-signing`.
- SignPath artifact configuration slug: `single-exe`.

Pushing to `main` does not trigger signing. It only runs the normal build and
test workflow.

## Supported Diagnostic Inputs

SmartLogAnalyzer supports the standard Intune Device Diagnostics ZIP layout,
including files named like:

```text
(N) RegistryKey HKLM_... export.reg
(N) Command windir_system32_Dsregcmd_exe_status output.log
(N) Command windir_system32_netsh_exe_advfirewall_show_allprofiles output.log
(N) FolderFiles ...
results.xml
```

English and French command outputs are supported for several parsers, including
DSRegCmd and firewall profiles.

## Recognized Error Code Families

| Range | Area |
| --- | --- |
| `0x8018xxxx` | MDM enrollment |
| `0x87D1xxxx` | Compliance and applications |
| `0xCAAxxxxx` | AAD/WAM authentication |
| `0x8007xxxx` | Windows system errors |
